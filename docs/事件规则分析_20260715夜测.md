## 二、规则算法(可直接落服务端)

预处理:
1. 标定:设备静止(v=0 持续>60s 且 |G|<50)时,滑动更新直立基准 ref = A 的中位数;
2. 每包计算 倾角 tilt = angle(A, ref),陀螺模长 |G|,加速度模长 |A|(静止≈16384)。

### R1 摔车检测(最可靠,建议先上线)

```text
确定摔倒: tilt ≥ 70° 连续 ≥ 2 包(约 10s)
疑似摔倒: tilt 45°~70° 连续 ≥ 2 包,且满足其一:
          ① 进入前 10s 内出现 |G| ≥ 2000 的尖峰
          ② 速度从 ≥3km/h 在一个采样间隔内突降为 0
恢复:     tilt < 40° 连续 ≥ 2 包
方向判定(倒地期间的 A):
  ax ≤ −8000 → 向右倒        (20:38 案例 ax=−10073)
  ax ≥ +8000 → 向左倒        (20:41 +15796,20:46 +12571)
  ay ≤ −8000 → 前扑/越过车头 (20:42 ay=−8400,可与左右组合)
```

回代验证:4 次摔车全中;单包 82° 的颠簸(21:23:34)被"连续 2 包"滤掉;
测试结束后 21:26~29 设备被搬动斜放(51~61°)因无尖峰、无速度突降,只会报"疑似"以下,不误报。
注意 ±8000 (≈0.5g) 的方向阈值基于本车安装角,换车需随 ref 一起标定。

### R2 停驻识别(红绿灯 / 等餐 / 驻车)

```text
停驻: v=0 且 tilt<40° 且 60s 净位移 <25m(复用平台漂移抑制的判定)
  30s ~ 3min  → 短停(红绿灯类)   本次 40s 红灯 ✓
  > 5min      → 驻车/等餐         本次 19min 等餐 ✓,且倾角/陀螺恒定可作"熄火驻车"佐证
```

### R3 颠簸路段

```text
移动中(v ≥ 4km/h)滚动 60s 计算 |A| 的标准差:
  std ≥ 3500 LSB(≈0.21g)→ 颠簸路段
辅助:单包 | |A| − 16384 | ≥ 8000(严重失重/冲击,如 |A|=3377)在窗口内出现 ≥2 次
```

回代:21:23 分钟 std=4215 唯一命中,正常骑行分钟最高 2941,间隔足够。

### R4 急刹车(当前数据只能做"骤停"级)

```text
相邻两包 Δv ≤ −10 km/h 且末速 < 5 km/h → 急刹
```

回代:20:44 命中(11.3→0);20:49 漏(刹车前已减速);正常停车(6~8km/h 滑行到 0)不误报。
电动车速度基数低,该规则天花板明显,**做严肃的急刹/碰撞检测必须提高采样率**(见第四节)。

### R5 坡道(暂不建议上规则)

骑行时姿态角包间抖动 ±15°,淹没坡度带来的 3~8° 偏移;GPS 高程趋势可见(−14m/3min)但常段无效。
可选方案:设备端输出 1~5s 平滑姿态角,或加气压计;否则先不做。

## 三、误报抑制要点(从本次数据学到的)

- 正常骑行瞬时倾角可达 55°(过坎/起步),**单包倾角绝不能直接报摔车**,必须持续 2 包以上;
- 停驻的倾角取决于撑脚架姿态(本次 5°~25°),恢复阈值 40° 要留出余量;
- 设备被人为搬动/斜放(21:26~29 持续 50~60°)靠"无尖峰、无速度突变"与真摔车区分;
- 陀螺静止零漂 <50,躺平后 |G|<35,"倒地后归于平静"是摔车与持续颠簸的本质区别。

## 四、对硬件方的采样需求(直接决定算法上限)

1. 恢复 **1 秒/包**;当前 4~5s 瞬时采样,碰撞(<1s)必漏,急刹只剩速度可用;
2. 六轴改为上报**窗口内峰值+均值**(而非瞬时值),或事件触发时前后 2s @100ms 补传;
3. 仍需书面确认 0xF1 前/后 6 字节实际含义与量程(本分析按"前=加速度计 16384/g、后=陀螺仪"推断,四次摔车物理自洽,基本坐实,但要固件侧盖章);
4. 高程字段大量时段为 0,请确认是否只在 3D 定位时有效。
