# 车机设备接入文档(JT/T 808 · 含陀螺仪扩展)

> 平台版本 v1 · 适用于按 JT/T 808 协议上报的车载终端 / 定位盒子

## 一、接入地址

| 项目 | 地址 |
|---|---|
| JT808 接入(TCP) | `186.244.238.6:18808` |
| 轨迹地图页面 | `http://186.244.238.6:18209/` |
| 数据查询 API | `http://186.244.238.6:18209/api/devices` |

设备端只需要把"主服务器地址/端口"配置为上表 TCP 地址,**通信方式 TCP,上报频率 1 秒/包**。

## 二、协议要求

- 标准 **JT/T 808-2013 或 808-2019**,平台按消息体属性中的版本标识自动识别,两版都可直接接入;
- 帧格式:`0x7e` 定界 + `0x7d` 转义 + 尾部异或校验(标准要求,不满足会被丢帧);
- 终端手机号(消息头 BCD 字段)即平台侧的**设备唯一 ID**,页面按它展示,请保证每台设备唯一;
- 暂不支持分包消息(0x0200 单包远小于上限,无影响);加密位须为 0(不加密)。

### 消息交互流程

```text
设备                              平台
 │ ── 0x0100 终端注册 ──────────▶ │
 │ ◀───── 0x8100 注册应答(含鉴权码) │   结果=0 成功
 │ ── 0x0102 终端鉴权(鉴权码) ───▶ │
 │ ◀───── 0x8001 平台通用应答 ───── │   结果=0 成功
 │ ── 0x0200 位置汇报(1秒/包) ──▶ │
 │ ◀───── 0x8001 平台通用应答 ───── │   每包都会应答
 │ ── 0x0002 心跳(可选) ─────────▶ │   有位置流时可不发
```

说明:
- 鉴权码由平台在注册应答中下发(格式 `AUTH+手机号后6位`),设备重连后可直接用旧鉴权码鉴权,不必重复注册;
- 未注册直接鉴权也会被接受(演示环境宽松策略);
- 其他消息(注销、参数应答等)平台统一回 0x8001,不会断连接。

## 三、位置汇报 0x0200

消息体按国标:报警标志(4) + 状态(4) + 纬度(4) + 经度(4) + 高程(2) + 速度(2, 0.1km/h) + 方向(2) + 时间(BCD6, 东八区 YYMMDDhhmmss),后接附加信息列表。

要求与说明:

- 经纬度用 **WGS-84 原始 GPS 坐标**,单位 1e-6 度;百度地图纠偏由平台完成,设备端不要自行转换坐标系;
- 状态位 bit1(定位有效)请如实置位;南纬 bit2、西经 bit3 支持;
- 速度单位 0.1 km/h,方向 0~359(正北为 0)。

### 附加信息:0xF1 陀螺仪扩展(本项目扩展项)

| 附加信息 ID | 长度 | 说明 |
|---|---|---|
| `0xF1` | 12 字节 | 三轴角速度 + 三轴加速度,全部 **int16 大端有符号** |

字节布局:

| 字节 | 内容 | 说明 |
|---|---|---|
| 1~2 | X 轴角速度 | 左右侧翻/倾斜速度,正负代表方向 |
| 3~4 | Y 轴角速度 | 前后俯仰/颠簸速度 |
| 5~6 | Z 轴角速度 | 转向 |
| 7~8 | X 轴加速度 | 车身左右受力 |
| 9~10 | Y 轴加速度 | 前后惯性(急刹车) |
| 11~12 | Z 轴加速度 | 垂直受力,静止约 1000mG |

平台同时兼容解析标准附加项:0x01 里程、0x30 信号强度、0x31 卫星数;
其他未知附加项会原样落库(hex),不会导致解析失败。

## 四、联调验证步骤

1. 设备配置服务器 `186.244.238.6:18808` 后上电;
2. 打开 `http://186.244.238.6:18209/`,左侧设备列表应出现设备号且绿点在线(30 秒无包判离线);
3. 点选设备 → 默认"实时跟踪",地图轨迹应每秒延伸;右下角遥测面板显示车速/方向/陀螺仪六轴;
4. 切"历史轨迹"可按时间段回放;
5. 接口核验:`GET http://186.244.238.6:18209/api/devices/{设备号}/latest`,能看到最新点位与陀螺仪字段。

## 五、参考实现与常见问题

- `scripts/simulator.py` 是一份完整的终端侧参考实现(注册/鉴权/1Hz 位置/0xF1 组包),字节级可对照;
  本地验证:`uv run python scripts/simulator.py --host 186.244.238.6 --device <11位号码>`
- 平台侧对**每一帧上行/下行原始报文都有落盘**(`data/raw/jt808-日期.log`,hex 原文),
  联调有疑问时把设备号和时间点发给平台方,可逐字节比对;
- **设备连上就断**:检查尾部异或校验范围(消息头第一个字节到消息体最后一个字节)和 0x7d 转义顺序;
- **设备在线但地图无轨迹**:检查状态位 bit1 是否置了"已定位",未定位的点平台照收,但经纬度为 0 会画在地图外;
- **轨迹偏移几百米**:说明设备端上报的不是 WGS-84(可能已转过 GCJ-02),请改回原始 GPS 坐标;
- **时间显示差 8 小时**:0x0200 的 BCD 时间按国标应为东八区本地时间,不是 UTC。
